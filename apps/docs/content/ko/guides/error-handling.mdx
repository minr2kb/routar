# 에러 처리

## 에러 타입

| 에러 | 패키지 | throw 시점 |
|-------|---------|------------|
| `ValidationError` | `@routar/core` | 요청 또는 응답 검증 실패 |
| `HttpError` | `@routar/core` | 서버가 비-2xx 상태 반환 (fetch executor만) |
| `TimeoutError` | `@routar/core` | `createFetchExecutor`의 `timeout` 초과 |
| `AxiosError` | `axios` | Axios의 네트워크 또는 HTTP 에러 |

## 에러 처리

```ts
import { TimeoutError, ValidationError, HttpError } from '@routar/core'

try {
  await todoApi.create({ body: { title: '' } })
} catch (err) {
  if (err instanceof ValidationError) {
    console.log(err.message, err.cause)
  }
  if (err instanceof HttpError) {
    console.log(err.status, err.statusText, err.body)
  }
  if (err instanceof TimeoutError) {
    console.log(err.ms)
  }
}
```

## executor 타입별 분기

```ts
import { HttpError } from '@routar/core'
import { isAxiosError } from 'axios'
import { HTTPError } from 'ky'

try {
  await todoApi.getDetail({ path: { id: 1 } })
} catch (err) {
  if (err instanceof HttpError) {
    console.log(err.status, err.body)
  } else if (isAxiosError(err)) {
    console.log(err.response?.status, err.response?.data)
  } else if (err instanceof HTTPError) {
    console.log(err.response.status, await err.response.json())
  }
}
```

## 트랜스포트 간 에러 정규화

```ts
import { definePlugin, HttpError } from '@routar/core'
import { isAxiosError } from 'axios'

const errorNormalizer = definePlugin({
  name: 'error-normalizer',
  onError: async (err) => {
    if (isAxiosError(err)) throw new HttpError(err.response?.status ?? 0, err.message)
    throw err
  },
})
```

## 서버 에러만 재시도

`retry`는 `createFetchExecutor`의 옵션입니다:

```ts
createFetchExecutor('https://api.example.com', {
  retry: {
    count: 3,
    shouldRetry: (err) => {
      if (err instanceof HttpError) return err.status >= 500
      return true
    },
  },
})
```
