# `@routar/react-query`

## `createQueries(api, options?)`

routar API 클라이언트에서 타입이 적용된 `queryOptions` / `mutationOptions` 팩토리를 파생합니다. 라우터를 다시 전달할 필요가 없습니다 — `createApi`가 클라이언트의 `$router` 프로퍼티에 라우터를 실어두고 `createQueries`가 이를 복원합니다. 반환 객체는 라우터의 형태를 그대로 반영합니다 — GET 엔드포인트는 쿼리 접근자, GET이 아닌 엔드포인트는 뮤테이션 접근자가 됩니다.

```ts
import { createQueries } from '@routar/react-query'
import { todoApi } from './todo'

export const todoQuery = createQueries(todoApi)
```

### 파라미터

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `api` | `ApiClientWithRouter` (`createApi` 반환) | `createApi`로 생성된 타입 지정 클라이언트; 소스 라우터를 `$router`에 실고 있음 |
| `options.key` | `string \| string[]` | 루트 키 세그먼트 재정의 (기본값: 라우터 prefix에서 파생) |
| `options.defaults` | `Record<string, QueryOptions \| MutationOptions>` | 엔드포인트 이름을 키로 하는 per-endpoint 기본 옵션. per-call 옵션 **이전에** 병합됩니다 (per-call이 우선). 최상위 엔드포인트에만 적용; 뮤테이션 엔드포인트의 경우 값은 `invalidates`를 제외한 뮤테이션 옵션. |

### 에러 타이핑

쿼리/뮤테이션 결과의 `error`는 TanStack의 `DefaultError`로 타입이 지정됩니다. 프로젝트 전체에서 `HttpError`로 좁히려면, TanStack의 `Register` 인터페이스를 한 번만 확장하세요 — `createQueries` 변경 없이 접근자가 자동으로 반영합니다:

```ts
import type { HttpError } from '@routar/core'
declare module '@tanstack/react-query' {
  interface Register { defaultError: HttpError }
}
```

### 쿼리 접근자 (GET 엔드포인트)

각 쿼리 접근자는 호출 가능하며 TanStack `queryOptions` 객체를 반환합니다:

```ts
// 시그니처: (params?, queryOptions?) => queryOptions
todoQuery.getList()
todoQuery.getList({ query: { done: true } }, { staleTime: 60_000 })
todoQuery.getDetail({ path: { id: '1' } })
```

**`.queryKey(params?)`** — 해당 접근자의 브랜드 쿼리 키를 반환합니다:

```ts
todoQuery.getList.queryKey()                        // ["todos", "getList"]
todoQuery.getDetail.queryKey({ path: { id: '1' } }) // ["todos", "getDetail", { path: { id: "1" } }]
```

### 뮤테이션 접근자 (GET이 아닌 엔드포인트)

각 뮤테이션 접근자는 TanStack `mutationOptions` 객체를 반환합니다:

```ts
// 시그니처: (mutationOptions?) => mutationOptions
todoQuery.create()
todoQuery.create({ invalidates: [todoQuery.getList.queryKey()] })
todoQuery.update({ onSuccess: () => { /* ... */ } })
```

**`.mutationKey`** — 해당 접근자의 브랜드 뮤테이션 키:

```ts
todoQuery.create.mutationKey  // ["todos", "create"]
```

### 도메인 키 (`$key`)

접근자 객체의 모든 레벨에서 `$key`를 제공합니다 — 해당 도메인 또는 하위 도메인의 루트 세그먼트입니다:

```ts
todoQuery.$key         // ["todos"]
userQuery.posts.$key   // ["api", "v1", "users", "posts"]

// 전체 도메인 무효화:
qc.invalidateQueries({ queryKey: todoQuery.$key })
```

### `invalidates` 옵션

뮤테이션 접근자의 `invalidates` 옵션에 쿼리 키 배열을 전달합니다. 이 키들은 `mutation.meta`에 저장되고 뮤테이션 성공 후 `routarMutationCache`에서 처리됩니다.

```ts
useMutation(
  todoQuery.create({
    invalidates: [todoQuery.getList.queryKey(), todoQuery.$key],
  }),
)
```

`QueryClient`에 `routarMutationCache`가 배선되어 있어야 합니다 (아래 참조). 배선 없이는 `invalidates`가 아무것도 하지 않습니다. 개발 환경에서는 `routarMutationCache`가 배선되지 않은 상태에서 뮤테이션이 `invalidates`를 선언하면 라이브러리가 일회성 `console.warn`을 출력합니다.

좁은 범위의 무효화를 권장합니다 — 실제 영향받는 특정 키를 지정하세요(예: `todoQuery.getList.queryKey()`). `todoQuery.$key`는 도메인 전체를 무효화하는 뮤테이션에만 예약하세요; 도메인 루트 아래의 모든 활성 쿼리를 리페치하므로 비용이 클 수 있습니다.

## `routarMutationCache(getClient)`

`mutation.meta`에 저장된 `invalidates`를 처리하는 `MutationCache` 팩토리입니다. `QueryClient` 생성 시 한 번만 배선하세요.

```ts
import { QueryClient } from '@tanstack/react-query'
import { routarMutationCache } from '@routar/react-query'

let queryClient: QueryClient
queryClient = new QueryClient({
  mutationCache: routarMutationCache(() => queryClient),
})
```

### 파라미터

| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `getClient` | `() => QueryClient` | `QueryClient` 인스턴스 getter (생성 시점의 순환 참조 방지) |

### 동작

모든 뮤테이션 성공 후, `routarMutationCache`는 `mutation.meta.invalidates`를 읽고 각 항목에 대해 `queryClient.invalidateQueries`를 호출합니다. `meta`에 `invalidates`가 없는 뮤테이션은 영향을 받지 않습니다.

`invalidates`와 기본 `onSuccess` 콜백을 함께 사용할 수 있으며 둘 다 실행됩니다.
