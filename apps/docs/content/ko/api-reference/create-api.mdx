# `createApi(executor, router)`

완전히 타입이 지정된 API 클라이언트를 생성합니다.

```ts
const todoApi = createApi(executor, todoRouter)

await todoApi.getDetail({ path: { id: 1 } })
await todoApi.create({ body: { title: 'buy milk' } })
```

## 오버로드

| 형태 | 사용법 |
|------|-------|
| `createApi(executor, router)` | 사전 빌드된 라우터 |
| `createApi(executor, prefix, endpoints)` | URL 접두사가 있는 인라인 |
| `createApi(executor, endpoints)` | 접두사 없는 인라인 |

## 파라미터

| 파라미터 | 타입 | 설명 |
|-----------|------|-------------|
| `executor` | `Executor` | HTTP 트랜스포트 |
| `router` | `RouterDef \| RouterEndpoints` | 라우터 또는 인라인 엔드포인트 |
| `prefix` | `string` (선택) | URL 접두사 |

## 요청 파라미터

```ts
await api.getDetail({ path: { id: 1 } })
await api.getList({ query: { page: 2 } })
await api.create({ body: { title: 'buy milk' } })
```

## per-call 옵션

두 번째 인자는 `AbortSignal`(하위호환) 또는 옵션 객체 `{ signal?, headers?, timeout? }`를 받습니다:

```ts
// AbortSignal로 취소
const controller = new AbortController()
await todoApi.getList({}, controller.signal)
controller.abort()

// per-call 헤더 + 타임아웃
await todoApi.create(
  { body: { title: 'buy milk' } },
  {
    headers: { 'Idempotency-Key': key }, // executor 기본 헤더보다 우선 머지
    timeout: 30_000,                      // 초과 시 TimeoutError
    signal: controller.signal,
  },
)
```

- **`headers`**는 요청에 시드되어 executor 기본 헤더보다 우선합니다. 플러그인 `onRequest`는 이후 실행되어 동일 키 충돌 시 우선합니다(인증 등 cross-cutting 정책).
- **`timeout`**은 `TimeoutError`로 요청을 중단하며 코어 레벨에서 적용되어 모든 executor(fetch/Axios/ky/custom)에서 동작하고 executor 레벨 타임아웃과 조합됩니다(먼저 발화한 쪽 우선).

## 검증 모드

`validate`는 요청/응답 스키마 실행 여부를 단계별로 제어합니다. 각 단계는 `true`(기본 — 검증 후 실패 시 throw), `false`(skip), `'warn'`을 받습니다:

```ts
// 프로덕션 장애 없이 응답 drift 관찰
const api = createApi(executor, todoRouter, {
  validate: { request: true, response: 'warn' },
  onValidationError: (err, ctx) => {
    // ctx: { kind: 'request' | 'response', method, url, data }
    Sentry.captureException(err, { extra: ctx })
  },
})
```

`'warn'` 모드는 parse 실패 시 **raw** 데이터를 그대로 통과시키고(장애 없음) throw 대신 `onValidationError(error, context)`를 호출합니다 — 전체 검증과 `validate: false` 사이의 drift 관찰 중간 지대입니다. `onValidationError`는 `true`(throw)나 `false`(skip)에서는 호출되지 않습니다.

## SSR/CSR 패턴

```ts
export const todoApi       = createApi(clientExecutor, todoRouter) // CSR
export const todoServerApi = createApi(serverExecutor, todoRouter) // SSR
```
