# Middleware

Middlewares are `(opts, next) => Promise<unknown>` functions applied in declaration order. Pass them when creating an executor.

```ts
import { createExecutor, withTimeout, withRetry, withLogger, defineMiddleware } from '@routar/core'

const executor = createExecutor(transport, [
  withTimeout(8_000),
  withRetry(3, { shouldRetry: (err) => !(err instanceof HttpError && err.status < 500) }),
  withLogger({ log: (msg, data) => logger.debug(msg, data) }),
])
```

## Built-in middleware

### `withRetry`

```ts
withRetry(count, options?)
```

Retries failed requests up to `count` times.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `count` | `number` | — | Maximum retry attempts |
| `shouldRetry` | `(err: unknown, attempt: number) => boolean` | `() => true` | Return `false` to skip retry for specific errors. `attempt` is zero-indexed (0 = first failure). |

```ts
withRetry(3)
withRetry(3, { shouldRetry: (err) => !(err instanceof HttpError && err.status < 500) })
```

### `withTimeout`

```ts
withTimeout(ms)
```

Aborts the request if no response arrives within `ms` milliseconds. Uses `AbortSignal` internally and merges with any existing signal passed to the executor.

```ts
withTimeout(5_000)  // 5 seconds
```

When the timeout fires, the rejection reason is a `TimeoutError` (exported from `@routar/core`), so you can distinguish a timeout from a user-initiated `AbortSignal` cancellation:

```ts
import { TimeoutError } from '@routar/core'

try {
  await api.getList()
} catch (err) {
  if (err instanceof TimeoutError) {
    console.log(`Timed out after ${err.ms}ms`)
  }
}
```

### `withLogger`

```ts
withLogger(options?)
```

Logs each request with method, URL, and duration.

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `log` | `(msg: string, data?: unknown) => void` | `console.log` | Custom log function |

```ts
withLogger()
withLogger({ log: (msg, data) => logger.debug(msg, data) })
```

## Custom middleware

Use `defineMiddleware` to create reusable middleware:

```ts
import { defineMiddleware } from '@routar/core'

const withCorrelationId = defineMiddleware((opts, next) =>
  next({ ...opts, headers: { ...opts.headers, 'X-Request-Id': crypto.randomUUID() } })
)

const withAuth = defineMiddleware(async (opts, next) => {
  const token = await getToken()
  return next({ ...opts, headers: { ...opts.headers, Authorization: `Bearer ${token}` } })
})
```

The `next` function continues the chain. Always return its result.

## Error normalizer (example)

routar lets each transport throw its own error type. If you need a unified error across fetch and axios, add a normalizer middleware:

```ts
import { isAxiosError } from 'axios'
import { HttpError } from '@routar/fetch'

const withErrorNormalizer = defineMiddleware(async (opts, next) => {
  try {
    return await next(opts)
  } catch (err) {
    if (isAxiosError(err)) throw new HttpError(err.response?.status ?? 0, err.message)
    throw err
  }
})
```
